Introduction

This sample code provides the Windows Phone game development with MVVM pattern and Local database for storing the player and score card information.

This BALL app is available in Windows Phone Store

Building the Sample

To run this code sample in development machine, machine should have Visual Studio 2010 or higher with Windows Phone 7.1 and SQL Server.

Description

This Sample code has two types for ball games.

  1. Ball Counter - Player has to select the moving balls - score will calculate based on how fast it complete.
  2. Ball Balance - Player has to balance the ball and place it in boat object - score will calculate based on how fast it complete.   

Local Database has created from SQL Server database using the WINDOWS PHONE LOCAL DATABASE extension.

Database

Database structure as below:

 Table Descriptions 

  1. Settings - Store audio setting 
  2. Players - Store the player name.
  3. ScoreCard - Store score, Level and no. attempts information for a player.
  4. Games  - This will have the types of game - not used in the code.

 

Solution

 This Balls solutions has 10 projects with four different categories, see the solutions structure below:


 Project Descriptions

  1. Balls.Data - This project holds the database object and take care the database creation & reset the  database. This project is created using WINDOWS PHONE LOCAL DATABASE extension (http://visualstudiogallery.msdn.microsoft.com/7678f224-48cf-421e-80a8-85ace0a7afba)
  2. Balls.Business.Services - This project is intermediate between the database and UI projects.
  3. Balls.Business.Components - This project is creates the game components like Balls, Bars, Game Text  etc.
  4. Balls.Common.Infrastructure - This project take care the all infrastructure classes like base classes, control classes, converter classes etc.
  5. Balls.Common.Interfaces - This project holds all the View, ViewModel, GameComponent interfaces.
  6. Balls.Common.Models - This project having all the Model objects.
  7. XnaSystemComponent - This project helps to collection of game component update and draw. This code is created based on the Sl\XNA GAME COMPONENT SAMPLE (from code.microsoft).
  8. Balls.UI Balls.UILib Balls.UILibConents - This is the XNA 4.0 - Windows phone Game (4.0) project, it has game page with its viewmodel classes and also normal page with its viewmodel classes.

 public partial class CounterGameView : PhoneApplicationPage, ICounterGameView
    {
        public CounterGameView()
        {
            InitializeComponent();
        }

        public void SetViewModel(Common.Interfaces.Base.IBaseViewModel viewModel)
        {
            this.DataContext = viewModel;
        }

        protected override void OnNavigatedTo(System.Windows.Navigation.NavigationEventArgs e)
        {

            if (e.NavigationMode == NavigationMode.Back)
            {
                XAML.Shell.Navigate();
            }
            else
            {
                (new CounterGameViewModel(this)).LoadData(e.Uri.ToString());
                base.OnNavigatedTo(e);

            }
        }

        protected override void OnNavigatedFrom(System.Windows.Navigation.NavigationEventArgs e)
        {
            ((CounterGameViewModel)this.DataContext).UnloadContent();
            base.OnNavigatedFrom(e);

        }
    }
